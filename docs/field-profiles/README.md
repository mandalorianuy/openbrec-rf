# Field profiles

Un field profile describe una configuración validada para una misión, entorno y jurisdicción específicos. Es una capa posterior, no la autoridad de la Open Spec.

Actualmente no hay field profiles publicados: el estado real es `unverified`. Cuando exista evidencia, cada perfil debe enlazar su evidence pack, fijar componentes/configuración, límites ambientales, regulación revisada, roles, protocolo y vigencia. Cambiar cualquiera de esos elementos requiere reevaluar los claims afectados.

Una guía, simulación o prueba de banco no se convierte en field profile. Consultar [arquitectura documental](../DOCUMENTATION_ARCHITECTURE.md) y [evidence packs](../evidence-packs/README.md).
